1. Folder Structure

I'll start to explain what your download package comes with, and what you should look at and upload to your project. The downloaded package comes with some examples of how to get video in a site very quickly, you also have the scripts you'll need to import so you have the following:

  • Flash folder - This is where the flash files are located for the flash video fallback
  • Js folder - This folder contains the scripts you need to import for the script to work
  • Videos folder - This folder is just there for example purposes and you do not need to upload this to your project for it to work

You then have the html files which are examples on how to implement and call the script easily. In resume you should take to your project only the folders "flash" and "js", the rest will be modifications to your project's html or php file depending on which technologie you're using.

2.3 Using the script

If you followed all the scripts above, you can now use the script to add video to where you want, we've provided some usage examples in the download package:

  • full_example - This demonstrates how to use the script to add native video as background to your whole site
  • full_yt_example - This demonstrates how to use the script to add youtube video as background to your whole site
  • div_example - This demonstrates how to use the script to add youtube video as background to a div
  • div_control_example - This demonstrates how to use the script to add native video as background to a div and use the script api to control the video

So you call the script with the call video_background($jquery_object, options), in this case we're adding background to the whole page so to the body tag, but you can select any div.

So you have a lot of options to customize how your video functions, let's view what each one does:

  • position - The position css value for the video object to be created, make it "absolute" or "fixed", being that the latest makes it scroll with the page (used for fullpage backgrounds usually)
  • z-index - The z-index css property for the background video, usually set it to "-1" or "0", being that -1 will put it behind absolutely everything, even the background-color itself
  • video_ratio - If you want your video to fill all the available area (will crop some parts of the video potencially but looks terrific) set here the value resulting from the calculation width_of_the_video/height_of_the_video
  • loop - Set to true or false depending if you want the video to loop or not when it reaches the end
  • autoplay - Set to true or false depending if you want the video to start to play immediately or not
  • muted - Set to true or false depending if you want the video to start muted or not
  • mp4 - The path to the mp4 video format
  • webm - The path to the webm video format
  • ogg - The path to the mp4 video format
  • flv - The path to the mp4 video format
  • youtube - The id of the youtube video
  • start - Available for youtube videos only, sets the time in seconds you want the video to start at
  • fallback_image - The image to fallback to on mobile devices
  • sizing - The sizing you want the fallback image to have. Choose between "fill" or "adjust"
  • priority - Accepts "html5" or "flash", the choosen one will be given priority to use and fallback to the other if not available

3. Script Api

You have a lot of control functions you can call through the video_background you've created:

  • play() - Plays the video
  • pause() - Pauses the video
  • toggle_play() - Plays/Pauses the video according to it's state
  • mute() - Mutes the video
  • unmute() - Unmutes the video
  • toggle_mute() - Mutes/Unmutes the video according to it's state
  • hide() - Hides the video
  • show() - Shows the video
  • toggle_hide() - Hides/Shows the video according to it's state
  • rewind() - Rewinds the video to the start
